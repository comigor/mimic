# Repository Guidelines

## Project Overview

Mimic is a static, single-page web implementation of the Brazilian charades game *Imagem & Ação*, in Portuguese (pt-BR). It is a self-contained client app: one HTML file, one stylesheet, one JS file, and a per-category word database served as JSON. No backend, no build, no package manager.

## Architecture & Data Flow

- `index.html` defines six sibling `<section class="screen">` panels — `home`, `board`, `reveal`, `timer`, `timeup`, `win` — toggled by adding/removing `.active`. Only one screen is visible at a time (`app.js:552` `show(name)`).
- `app.js` is a single top-level IIFE in `"use strict"` mode (`app.js:1`, `app.js:855`). All game logic lives inside; nothing is exported except `window.MIMIC_WORDS` (the loaded card pool).
- A central mutable `state` object (`app.js:85`) holds teams, board, current card, timer, audio, and animation flags. DOM references are cached once into `els` (`app.js:42`) and `screens` (`app.js:33`).
- Game flow: home → (start) → **board** → roll dice / land on category → **reveal** (cover → show card) → **timer** → either *acertou* (auto-roll, advance pawn, next team) or *time up* → back to board. Reaching tile 35 triggers the **win** screen.
- Board is a rectangular spiral path on a 6×6 grid (36 tiles), generated by `generateSpiralPath` (`app.js:105`) and rendered to a `<canvas>` by `drawBoard` (`app.js:206`). Tile 0 is `T` (Todos), tile 35 is `FINISH`, tiles 1–34 cycle `P, O, A, D, L, T`.
- Word pool is built at startup by `loadWords()` (`app.js:818`): `fetch("words/<cat>.json")` for each of `p,o,a,d,l,t`, dedupe by `word|category`, expose as `window.MIMIC_WORDS = [{ word, category, allPlay }]`. `pickCard()` (`app.js:531`) filters by `state.forcedCategory` (board mode) or active checkboxes (free play), and keeps a rolling `state.recent` list (cap = `min(20, pool/2)`) to avoid immediate repeats.
- Audio: `primeAudio()` lazily creates a `WebAudio` `AudioContext` on first user gesture; `playAlarm()` schedules square-wave beeps via `beep()` (`app.js:708`). No audio assets are bundled.
- Persistence: `localStorage` keys `mimic.teamCount`, `mimic.timer`, `mimic.cats` (`app.js:728`, `app.js:789–814`).

## Key Directories

- `/` — all source. `index.html` is the entry; `app.js` is the only script tag (`index.html:140`); `style.css` is the only stylesheet.
- `words/` — six JSON files, one per category letter (`p.json`, `o.json`, `a.json`, `d.json`, `l.json`, `t.json`). Each is a flat array of `[word]` or `[word, 1]` entries; the trailing `1` flags `allPlay: true` (every team plays simultaneously). Files in the `t.json` (Todos) bucket are implicitly `allPlay`. ~360–370 entries per file (~2200 total).
- `.github/workflows/deploy.yml` — GitHub Pages deploy.

## Development Commands

There is **no build, no package manager, no test runner**. Workflow:

```sh
# Serve locally (any static server works; required because app.js uses fetch()).
python3 -m http.server 8000
# or
npx --yes serve .
```

Opening `index.html` via `file://` will fail to load words (CORS on `fetch`). Always serve over HTTP.

No `npm install`, `lint`, `format`, or `test` commands exist. Do not introduce a build step or a package manager unless explicitly requested.

## Code Conventions & Common Patterns

- **Single IIFE, strict mode.** All new code goes inside the existing IIFE in `app.js`. No ES modules, no `import`/`export`, no globals other than `window.MIMIC_WORDS`.
- **`const` by default, `let` for reassignment**, never `var`. Mixed function styles: named `function` declarations for module-level helpers, arrow callbacks for event handlers and timers.
- **camelCase** for variables and functions; **UPPER_SNAKE_CASE** for module constants (`TILE_SEQUENCE`, `BOARD_COLS`, `CAT_COLORS`).
- **DOM access**: cache references into `els` / `screens` at startup; do not call `document.getElementById` ad hoc inside handlers.
- **Screen transitions** go through `show(name)` — never set `.active` manually.
- **State mutation** is direct on the singleton `state` object; no reducers, no events, no observers. Side-effecting helpers (`startTeamTurn`, `nextTurn`, `showCardState`) read and write `state` freely.
- **Async**: `loadWords` uses `fetch` + `Promise.all` + `.then` chains (no `async/await`). Timers use `setInterval` / `setTimeout` directly; `state.intervalId` tracks the running timer so it can be cleared in `stopTimer` (`app.js:623`).
- **Canvas redraw safety**: iOS Safari can drop the canvas buffer. The codebase deliberately over-redraws via `redrawBoardSoon` (`app.js:407`), a 1.5s interval safety net (`app.js:425`), `ResizeObserver`, `resize`, `pageshow`, `focus`, and `visibilitychange` listeners (`app.js:755–786`). Preserve this redundancy when touching board rendering.
- **Categories** are single-letter codes (`P`, `O`, `A`, `D`, `L`, `T`); their human labels live in `CATEGORY_LABELS` (`app.js:4`) and colors in `CAT_COLORS` / CSS vars `--cat-X`. Adding a category requires updating: `CATEGORY_LABELS`, `CAT_COLORS`, `TILE_SEQUENCE`, `CAT_TEXT_DARK` (if light tile), `index.html` checkboxes, `style.css` `--cat-X` + `.cat-X`, and adding `words/<x>.json` plus extending the `cats` array in `loadWords` (`app.js:820`).
- **CSS** uses CSS custom properties on `:root` (`style.css:1`) for theme + categories + team colors. No preprocessor, no framework, no utility classes — plain class selectors with mobile-first sizing (`max-width: 540px` per screen; `max-width: 420px` for the board canvas).
- **Language**: all user-facing strings are pt-BR. Keep new UI strings in Portuguese; identifiers and comments are English.

## Runtime / Tooling Preferences

- **Target**: modern evergreen browsers (Chrome, Safari ≥ 14, Firefox). Specifically tested for iOS Safari quirks (see redraw safety net above).
- **No runtime/framework** — vanilla JS, DOM, Canvas 2D, WebAudio. Do not add React/Vue/Svelte/jQuery/lodash/etc.
- **No bundler, no transpiler.** Code must run as-is in the browser; use only language features supported without transpilation (optional chaining, nullish coalescing, `?.()`, `roundRect` with manual fallback at `app.js:187` are all fine).
- **No npm/Node tooling.** Do not introduce `package.json`, `tsconfig`, ESLint, or Prettier configs unless explicitly asked.
- **Local serving** requires a static HTTP server (see Development Commands). `fetch` will not work from `file://`.

## Testing & QA

- There is no automated test suite, no test framework, no CI test step.
- Verify changes manually in a browser served over HTTP:
  - Home screen loads, footer shows `N palavras` (not "Erro ao carregar palavras").
  - Each of 2/3/4 team configs starts a game; pawns appear on tile 1.
  - Dice roll animates, pawn advances tile-by-tile, lands on the matching-category card.
  - Reveal cover hides the card until *Mostrar carta*; timer counts down; *Acertou!* auto-rolls; *Parar* hands off without rolling; alarm sounds at 0.
  - Free play mode (*Modo livre*) ignores the board and respects the checkbox filter.
  - Settings persist across reloads (team count, timer, active categories).
- If you change rendering, exercise resize, tab-switch, and a backgrounded-then-foregrounded iOS Safari path — the board must repaint.
- The deploy workflow uploads the repo root verbatim as a Pages artifact (`.github/workflows/deploy.yml:37`). Any file you add ships to production.
